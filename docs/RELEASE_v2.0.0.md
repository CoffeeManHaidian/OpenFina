# OpenFina v2.0.0 Release Notes

发布日期：2026-04-22

## 发布定位

OpenFina v2.0.0 是一次架构级更新。本版本将系统从早期的单一公司/年度凭证库模式，升级为“用户库 + 账套库”的两层数据库结构，为多企业、多会计年度、多账套使用场景打下基础。

本版本暂不兼容旧版历史数据。旧测试数据可抛弃，首次运行会按新结构初始化用户库和账套库。

## 已实现功能

### 应用启动与登录

- 新增独立启动入口 `app/bootstrap.py`，负责应用初始化、自动登录恢复和登录页/主界面分发。
- 支持用户注册、登录、记住登录和自动登录。
- 自动登录失效时会清理旧设置，并回退到普通登录页。
- 注册用户时必须创建初始账套。
- 登录后根据用户绑定的账套进入当前账套。
- 一个用户绑定多个账套时，支持选择本次登录账套，并可设为默认账套。

### 用户库与账套库

- 用户库 `users.db` 只保存用户、账套目录和用户账套关系。
- 用户库包含：
  - `users`
  - `booksets`
  - `user_booksets`
- 每个账套使用独立 SQLite 数据库文件。
- 账套按“企业 + 会计年度 + 账套 ID”隔离，避免多个企业或多个年度的数据混淆。
- 账套库包含：
  - `bookset_info`
  - `subjects`
  - `voucher_master`
  - `voucher_details`
- 新建账套时会自动初始化账套元数据、会计科目和凭证表。

### 会计科目

- `source/subject.json` 作为新账套初始化模板保留。
- 新账套创建时会把模板科目导入到账套库的 `subjects` 表。
- 运行时科目查询、科目选择、科目新增和删除均基于当前账套库。
- 不同企业、不同年度账套可以维护各自独立的科目数据。

### 凭证录入

- 支持凭证录入窗口。
- 支持凭证字、凭证号、业务日期、会计日期、摘要、科目、借方金额、贷方金额录入。
- 支持 `F2` 打开科目选择窗口。
- 支持借贷合计自动计算。
- 支持保存凭证主表和凭证明细。
- 凭证底部支持手动填写：
  - 制单
  - 审核
  - 过账
  - 出纳
  - 核准
- 制单、审核、过账显示姓名与登录账号分离，便于业务留痕和权限校验扩展。

### 凭证查询

- 主界面“凭证查询”现在先打开凭证列表弹窗。
- 凭证列表按当前会计期间显示，例如 `2026年04期`。
- 支持点击会计期间标签切换期间。
- 列表表格展示：
  - 凭证号
  - 业务日期
  - 凭证字
  - 摘要
  - 状态
  - 制单人
  - 审核/过账信息
- 支持菜单栏筛选：
  - 按摘要筛选
  - 按会计科目筛选
  - 切换会计期间
  - 清除全部筛选
- 双击凭证列表中的凭证，可以打开只读凭证详情。

### 双敲审核

- 主界面已接入“双敲审核”功能。
- 支持按月份选择凭证。
- 支持查看凭证明细。
- 支持审核和取消审核。
- 制单人与审核人不能为同一人。
- 已过账凭证不能取消审核。
- 取消审核按审核操作账号校验，防止非原审核账号撤销。

### 凭证过账

- 主界面已接入“凭证过账”功能。
- 支持按月份选择凭证。
- 未审核凭证不能过账。
- 已审核凭证可以过账。
- 已过账凭证可由原过账账号取消过账。
- 取消过账按过账操作账号校验。

### 凭证汇总

- 支持凭证汇总窗口。
- 支持按期间、科目范围和科目级别生成汇总数据。
- 支持切换汇总条件，不需要关闭窗口重新打开。
- 汇总基于当前账套库，不再从公司名和年度推导数据库。
- 科目名称映射来自当前账套库的科目表。

### 主界面与 UI

- 主界面已调整为 v2.0 风格。
- 标题栏保留原有右侧最小化、最大化、关闭布局。
- 主界面取消地址导航栏和多标签页设计。
- 功能区、子功能区、明细功能区向上对齐。
- 已优化子功能和明细功能区域宽度，减少内容显示不全。
- 登录页、自动登录页、账套弹窗和主要业务窗口统一为更清爽的桌面应用风格。

### 日志与诊断

- 已补充应用启动、路径、数据库、登录、注册、账套、科目、凭证保存、审核、过账、汇总等关键日志。
- 已增加全局异常日志记录。
- 打包后若功能异常，可优先查看运行目录下的 `data/logs/`。

### 测试数据

- 新增 `scripts/seed_release_v2_data.py`。
- 可生成多用户、多企业、多年度、多状态凭证测试数据。
- 测试数据覆盖：
  - 未审核凭证
  - 已审核凭证
  - 已过账凭证
  - 零售、服务、制造、科技等不同业务场景

### 打包交付

- 支持 PyInstaller 绿色版打包。
- 支持单文件打包参数。
- 支持安装版打包脚本。
- 打包入口已切换到 `app/bootstrap.py`。
- 打包后的用户数据和日志写入运行目录下的 `data/`。

## 使用入口

开发环境启动：

```powershell
python app/bootstrap.py
```

绿色版打包：

```powershell
python scripts/build_exe.py
```

安装版打包：

```powershell
python scripts/build_full.py
```

测试数据生成：

```powershell
python scripts/seed_release_v2_data.py
```

## 已知限制

- 本版本不迁移旧版历史数据。
- 当前角色权限仍未完全系统化，审核和过账主要依赖登录账号做撤销校验。
- 账簿、完整报表、期末结账、现金流量等模块仍处于预留或早期阶段。
- 凭证汇总已有基础能力，但完整会计报表体系仍需后续版本继续完善。

## 推荐测试路径

1. 注册新用户并创建初始账套。
2. 使用多账套账号登录，确认可以选择账套。
3. 新增凭证并保存。
4. 打开凭证查询，按期间查看凭证列表。
5. 使用摘要和会计科目筛选凭证。
6. 双击凭证列表打开凭证详情。
7. 对未审核凭证执行双敲审核。
8. 对已审核凭证执行过账。
9. 打开凭证汇总并切换汇总条件。
10. 查看 `data/logs/` 中的运行日志。
